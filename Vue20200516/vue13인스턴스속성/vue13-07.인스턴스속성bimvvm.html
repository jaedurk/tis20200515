아키텍처 패턴의 이해 MVP : Net winform, swing , jQuery MVC : 스프링, angular MVVM : WPF, React [참고] MVVM 패턴 Model은 다른 데이터 소스(데이터베이스, 파일 또는 서버)에 액세스하는 책임을 집니다 View는 데이터의 상태를 반영하여 적절한 형식으로 데이터를 나타내고 사용자 상호작용과 이벤트를 수집합니다. ViewModel은 Model 이 소화 할 수 있는 형태로 View에서 Model로 필요한 모든 데이터를 전달함으로써 View와 Model 간의 통신을 처리 합니다. 유효성 검사는 ViewModel 컴포넌트에서 수행합니다. 이 패턴은 두 단계로 작동합니다. View는 ViewModel을 알고 있고, ViewModel의 프로퍼티를 업데이트하며, 뒤에서 발생하는 모든 변경 사항을 추적합니다. ViewModel은 View의 존재를 인식하지 못합니다. 비슷한 방식으로 Model 은 ViewModel의 존재를 모르고 오직 ViewModel 이 Model 에 액세스합니다. ViewModel 은 이벤트와 데이터를 Model 에 전달하며, 이러한 데이터는 Model 이 해석 할 수 있는 형태로 View에 의해 푸시 됩니다. ViewModel 은 Model 에 의해 생성된 모든 변경 사항을 추적하여 View 및 비즈니스 규칙에 따라 필요한 신호를 View로 푸시 합니다.
